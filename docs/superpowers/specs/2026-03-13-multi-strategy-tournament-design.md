# 멀티 전략 토너먼트 스캘퍼 설계 스펙

## 개요

빗썸 거래소에서 자동 코인 선별 → 멀티 전략 페이퍼 트레이딩 → 성과 기반 실매매 배분을 수행하는 스캘핑 시스템.

**핵심 목표:**
- 백테스트-실매매 괴리 문제 해결 (페이퍼 트레이딩 검증 단계)
- 하루 20~50건 활발한 매매
- 100만원 이하 소액 운용
- 자동 코인 선별, 자동 전략 전환

## 기술 스택

- 기존 인프라 유지: 단일 HTML 파일, 클라이언트 사이드 JavaScript
- 빗썸 API v1 (REST: 1분봉 캔들 + 실시간 호가)
- Web Worker 기반 백그라운드 타이머
- key.json으로 API 키 관리

## 아키텍처

```
[코인 스크리너] → [백테스트 엔진] → [페이퍼 트레이딩] → [실매매]
     │                                      ↕
     │                              [괴리 감시 대시보드]
     │                                      ↑
     └──────────────── [토너먼트 매니저] ────┘
```

### 모듈 구성

| 모듈 | 역할 | 입력 | 출력 |
|------|------|------|------|
| 코인 스크리너 | 매매 대상 코인 자동 선별 | 빗썸 전체 코인 시세 | 상위 5~10개 코인 리스트 |
| 전략 엔진 | 5개 전략 독립 신호 생성 | 1분봉 OHLCV | 매수/매도 신호 |
| 페이퍼 트레이딩 | 가상 체결로 전략 검증 | 신호 + 실시간 호가 | 가상 포지션/손익 |
| 토너먼트 매니저 | 전략 성과 평가 및 자금 배분 | 페이퍼 성과 데이터 | 실매매 허가 전략 목록 |
| 실매매 엔진 | 실제 주문 실행 | 승격된 전략 신호 | 실제 포지션/손익 |
| 괴리 대시보드 | 백테스트/페이퍼/실매매 비교 | 3자 성과 데이터 | 시각화 + 경보 |

## 모듈 1: 코인 스크리너

### 실행 주기
- 10분마다 빗썸 전체 코인 스캔

### 선별 기준

| 기준 | 조건 | 이유 |
|------|------|------|
| 24h 거래량 | 상위 30% | 체결 안정성 |
| 1분봉 변동성 (ATR) | 상위 50% (거래량 통과 코인 중) | 스캘핑 수익 기회 |
| 스프레드 | 0.15% 이하 | 슬리피지 최소화 |
| 스테이블코인 | 제외 (USDT, USDC, DAI, TUSD, BUSD, USDD, XAUT) | 변동성 없음 |
| 최소 가격 | 10원 이상 | 틱 사이즈 문제 방지 |

### 출력
- 상위 5~10개 코인 리스트 (거래량×변동성 점수 내림차순)
- 각 코인의 ATR, 거래량, 스프레드 메타데이터 포함

## 모듈 2: 전략 엔진

### 공통 규칙
- 시간 프레임: 1분봉
- 손절: -0.5%
- 최대 보유시간: 10분
- 트레일링 스탑: +0.3% 이후 활성화, -0.2% 트레일
- 재진입 대기: 동일 코인 손절 후 3분 쿨다운

### 전략 1: RSI 반전 스캘퍼

**컨셉:** 단기 RSI 극단값에서 역추세 진입, 빠른 익절

| 항목 | 값 |
|------|-----|
| 지표 | RSI(7) |
| 매수 조건 | RSI ≤ 25 이고 직전 봉 대비 RSI 상승 전환 |
| 매도 조건 | RSI ≥ 75 이고 직전 봉 대비 RSI 하락 전환 |
| 익절 | +0.4% 또는 RSI 50 도달 |
| 특징 | 횡보/약한 추세장에서 유리 |

### 전략 2: 볼린저 스퀴즈 브레이크아웃

**컨셉:** BB 밴드 수축 후 확장 돌파 시 추세 방향 진입

| 항목 | 값 |
|------|-----|
| 지표 | BB(20, 2), 밴드폭 |
| 스퀴즈 감지 | 밴드폭이 최근 50봉 중 하위 20% |
| 매수 조건 | 스퀴즈 상태에서 종가가 상단밴드 돌파 |
| 매도 조건 | 스퀴즈 상태에서 종가가 하단밴드 돌파 (숏 불가이므로 미사용) |
| 익절 | +0.5% 또는 밴드 중심선 복귀 |
| 특징 | 횡보 → 추세 전환 시 포착 |

### 전략 3: 거래량 급등 모멘텀

**컨셉:** 비정상적 거래량 급등과 가격 돌파가 동시에 발생할 때 진입

| 항목 | 값 |
|------|-----|
| 지표 | 거래량 20봉 평균, 가격 15봉 고가 |
| 매수 조건 | 거래량 ≥ 평균의 3배 AND 종가 > 15봉 최고가 |
| 필터 | RSI(14) < 70 (과매수 진입 방지) |
| 익절 | +0.6% 또는 거래량 평균 이하로 하락 시 |
| 특징 | 급등 초기 포착, 뉴스/이벤트 반응 |

### 전략 4: EMA 크로스 스캘퍼

**컨셉:** 단기 EMA 크로스오버로 미세 추세 전환 포착

| 항목 | 값 |
|------|-----|
| 지표 | EMA(5), EMA(13) |
| 매수 조건 | EMA(5) > EMA(13) 골든크로스 AND RSI(14) > 40 |
| 필터 | 현재가 > EMA(55) (중기 추세 상승 확인) |
| 익절 | +0.4% 또는 EMA(5) < EMA(13) 데드크로스 |
| 특징 | 추세장에서 유리 |

### 전략 5: 가격 액션 핀바

**컨셉:** 긴 아래꼬리 캔들 패턴 감지 후 반전 진입

| 항목 | 값 |
|------|-----|
| 지표 | 캔들 패턴 분석 |
| 핀바 조건 | 아래꼬리 ≥ 전체 봉 길이의 66% AND 몸통 ≤ 전체의 20% |
| 매수 조건 | 핀바 출현 AND 직전 3봉 하락 추세 AND 거래량 ≥ 평균 1.5배 |
| 익절 | +0.4% 또는 직전 고가 도달 |
| 특징 | 패턴 기반, 다른 지표와 독립적 |

## 모듈 3: 페이퍼 트레이딩

### 가상 체결 규칙 (보수적)

슬리피지와 체결 불확실성을 반영하여 실매매보다 불리하게 시뮬레이션:

| 항목 | 규칙 |
|------|------|
| 매수 체결가 | 신호 발생 시점 매도1호가 + 1틱 |
| 매도 체결가 | 신호 발생 시점 매수1호가 - 1틱 |
| 수수료 | 0.04% 양방향 (편도 0.04% × 2 = 0.08%) |
| 체결 지연 | 신호 발생 후 다음 봉 시가로 체결 (1봉 지연) |
| 부분 체결 | 미고려 (전량 체결 가정) |

### 데이터 기록

각 가상 매매에 대해 기록:
- 전략명, 코인, 진입/청산 시간, 진입/청산 가격
- 수익률, 보유시간
- 진입 사유 (어떤 조건 충족)
- 대응하는 백테스트 예상 수익률

## 모듈 4: 토너먼트 매니저

### 전략 평가 기준

최근 2시간 페이퍼 트레이딩 성과로 평가:

| 지표 | 가중치 | 설명 |
|------|--------|------|
| 순수익률 | 40% | 총 수익 / 총 투입 |
| 승률 | 30% | 수익 거래 / 전체 거래 |
| 손익비 | 20% | 평균 수익 / 평균 손실 |
| 매매 횟수 | 10% | 최소 건수 충족 여부 |

### 자금 배분 규칙

| 규칙 | 내용 |
|------|------|
| 승격 조건 | 페이퍼에서 최소 20건 이상 매매 완료 |
| 실매매 전략 수 | 상위 2개 전략만 |
| 전략별 최대 동시 포지션 | 1개 |
| 전체 최대 동시 포지션 | 2개 |
| 포지션 크기 | 총 자금의 40% (2포지션이면 각 20%) |
| 강등 조건 | 연속 3회 손절 시 페이퍼로 강등 |
| 재평가 주기 | 30분마다 |

## 모듈 5: 괴리 감시 대시보드

### 표시 항목

1. **전략별 성과 카드**
   - 페이퍼 수익률 / 실매매 수익률 / 괴리율
   - 승률, 평균 수익, 평균 손실
   - 현재 상태 (페이퍼 / 실매매 / 비활성)

2. **괴리율 차트**
   - 30분 / 1시간 / 24시간 단위
   - 전략별 백테스트 vs 페이퍼 vs 실매매 비교 라인차트

3. **실시간 로그**
   - 매매 신호 발생, 체결, 손익 실시간 표시
   - 전략 승격/강등 이벤트

### 경보 조건

| 경보 | 조건 | 액션 |
|------|------|------|
| 괴리 경고 | 백테스트-페이퍼 괴리율 > 20% | 해당 전략 자동 비활성화 |
| 연속 손실 | 전체 연속 5회 손절 | 모든 실매매 일시 중지 |
| MDD 초과 | 일일 MDD > -3% | 당일 매매 중단 |

## 실매매 안전장치

| 항목 | 값 |
|------|-----|
| 일일 최대 손실 | -3% (자금 대비) |
| 일일 최대 매매 횟수 | 60건 |
| 단일 포지션 최대 | 자금의 25% |
| API 오류 시 | 즉시 모든 매매 중단, 기존 포지션은 시장가 청산 |

## UI 레이아웃

단일 HTML 페이지, 상단에서 하단으로:

1. **헤더**: 총 자산, 일일 손익, 가동 시간
2. **코인 스크리너 패널**: 현재 선별된 코인 목록 + 점수
3. **전략 토너먼트 보드**: 5개 전략 카드 (성과, 상태, 순위)
4. **활성 포지션**: 현재 보유 중인 포지션 실시간 표시
5. **괴리 차트**: 백테스트/페이퍼/실매매 비교
6. **매매 로그**: 최근 거래 내역 테이블

## 구현 순서

1. 코인 스크리너
2. 전략 엔진 (5개 전략 + 공통 신호/청산 로직)
3. 백테스트 엔진 (전략별 과거 데이터 검증)
4. 페이퍼 트레이딩 엔진
5. 토너먼트 매니저
6. 실매매 연동
7. 괴리 대시보드
8. UI 통합
9. 통합 테스트 및 소액 실전 검증
