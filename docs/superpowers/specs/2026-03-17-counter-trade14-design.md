# Counter-Trade 14 설계 스펙

## 컨셉
코인별 백테스트 기반 최적 전략 자동 배정 + 마틴게일 물타기 시스템

## 전체 흐름

```
스크리너(자동) → 코인 추천 리스트 → 사용자 체크박스 선택/제외 →
선택 코인별 과거 데이터 백테스트(6개 전략) → 최적 전략 1개 자동 배정 →
실매매 (마틴게일 물타기 + 평단가 기준 익절)
```

## 1. 스크리너 + 코인 선택 UI

- 기존과 동일: KRW 마켓, 거래량/변동성 기준 상위 30개 추천
- 스테이블코인 제외 (USDT, USDC, DAI 등)
- **변경점**: 코인별 체크박스 UI 추가
  - 스크리너 결과 테이블에 체크박스 컬럼 추가
  - 사용자가 매매 대상 코인을 직접 ON/OFF
  - 선택된 코인만 백테스트 → 전략 배정 → 매매 대상
  - 선택 상태는 localStorage에 저장/복원
- **코인 목록 갱신 규칙**:
  - 활성 포지션이 있는 코인은 스크리너에서 사라져도 목록에 유지
  - 사용자가 선택했지만 스크리너에서 빠진 코인은 회색 표시, 선택 유지
  - 새로 등장한 코인은 기본 미선택 상태
- 스크리너 갱신 주기: 10분

## 2. 전략 풀 (6개 후보)

각 코인에 대해 6개 전략을 모두 백테스트하고 최고 점수 전략 1개를 배정한다.

### 2-1. RSI 반전
- **지표**: RSI(7)
- **매수**: RSI ≤ 25 AND RSI 상승 전환 (RSI[i] > RSI[i-1])
- **성격**: 과매도 반등 포착
- **적합**: 급락 후 반등이 잦은 코인

### 2-2. BB 브레이크아웃
- **지표**: Bollinger Bands(20, 2σ), RSI(14)
- **매수**: BB폭이 최근 50봉 하위 20% (스퀴즈) AND 가격이 상단밴드 돌파 AND RSI < 65
- **성격**: 변동성 확장 초기 포착
- **적합**: 횡보 후 폭발하는 코인

### 2-3. EMA 추세추종
- **지표**: EMA(5), EMA(13), EMA(55), RSI(14)
- **매수**: EMA5 > EMA13 골든크로스 AND RSI 45~62 AND 가격 > EMA55 AND EMA13 3봉 연속 상승
- **성격**: 추세 초기 진입
- **적합**: 방향성 있는 코인

### 2-4. 거래량 급등
- **지표**: 20봉 평균거래량, RSI(14)
- **매수**: 거래량 ≥ 평균 × 5 AND 양봉 AND 15봉 최고가 돌파 AND RSI < 70
- **성격**: 수급 급변 포착
- **적합**: 뉴스/이벤트 반응 코인

### 2-5. 핀바 반전
- **지표**: OHLC 캔들 구조, 20봉 평균거래량
- **매수**: 아래꼬리 ≥ 전체길이의 66% AND 몸통 ≤ 20% AND 직전 3봉 하락 AND 거래량 ≥ 평균 × 1.5
- **성격**: 캔들 패턴 기반 반전
- **적합**: 꼬리가 긴 변동성 코인

### 2-6. MACD 다이버전스 (신규)
- **지표**: MACD(12, 26, 9)
- **매수**: 최근 40봉 내 가격 저점 2개를 비교 — 가격은 저점 갱신 BUT MACD 히스토그램은 저점 미갱신 (상승 다이버전스) AND MACD 히스토그램 양전환 (음→양)
- **저점 탐지**: 최근 40봉에서 MACD 히스토그램이 음수→양수로 전환된 지점을 기준으로 직전 음수 구간의 최저점 2개를 비교
- **성격**: 추세 피로/전환 포착
- **적합**: 하락 추세 마무리 구간

## 3. 백테스트 기반 자동 전략 배정

### 3-1. 백테스트 실행
- **데이터**: 과거 6시간 1분봉 캔들 (fetchHistoricalCandles, BACKTEST_HOURS=6)
- **워밍업**: 처음 60봉은 지표 초기화 구간 — 신호 생성 제외
- **시뮬레이션**: 6개 전략 각각에 대해 매수/매도 신호를 시뮬레이션
- **수수료 반영**: 시뮬레이션 손익에 FEE_ROUND(0.08%) 차감 적용
- **결과 수집**: 전략별 수익률%(totalPnlPct), 승률(winRate), 최대단일손실%(maxLossPct), 매매횟수(trades)

### 3-2. 점수 산출 (정규화 기반)
```
// 6개 전략의 각 지표를 0~1로 정규화 (min-max scaling)
normPnl = (pnlPct - minPnl) / (maxPnl - minPnl)    // 수익률 정규화
normWR  = winRate                                     // 이미 0~1
normSafe = 1 - (|maxLossPct| - minLoss) / (maxLoss - minLoss)  // 안전성 정규화

score = normPnl × 0.5 + normWR × 0.3 + normSafe × 0.2
```
- 매매횟수가 0이면 점수 = -999 (배정 불가)
- 모든 전략 점수가 음수이면 해당 코인은 매매 대상에서 자동 제외
- 동일 전략이 두 코인에 배정될 수 있음 (제한 없음)

### 3-3. 배정 및 재평가
- 최고 점수 전략 1개를 해당 코인에 배정
- UI에 배정된 전략명 + 백테스트 성적 표시
- **재평가 주기**: 30분마다 재백테스트 → 전략 교체 가능
- 전략은 매수 신호만 담당, 매도(익절/손절/타임아웃)는 공통 로직으로 처리
- 전략 교체 시 기존 포지션은 유지 (공통 매도 로직으로 청산)

## 4. 마틴게일 물타기 시스템

### 4-1. 매수 단계

| 단계 | 매수금액 | 누적투자 | 트리거 |
|------|---------|---------|--------|
| 1차 (진입) | 사용자 지정 금액 (A) | A | 전략 매수 신호 |
| 2차 | A × 2 | A × 3 | 평단가 대비 -2% |
| 3차 | A × 4 | A × 7 | 평단가 대비 -2% |
| N차 | A × 2^(N-1) | A × (2^N - 1) | 평단가 대비 -2% |

- **매수금액**: UI 입력란에서 사용자가 1차 진입 금액 직접 설정 (최소 5,000원)
- **마틴게일 배수**: 자동 계산 (× 2^(N-1))
- **최대 단계**: 다음 단계 금액 > 가용잔고이면 자동 중단
- **가용잔고**: 총잔고 - 다른 포지션의 예약 자금 (남은 물타기 단계 금액 합)
- **물타기 트리거**: 평단가 대비 -2% 하락 시마다

### 4-2. 평단가 계산
```
avgPrice = 총투자금액 / 총보유수량
```
- 각 단계 매수 시 체결가/수량을 누적하여 평단가 갱신

### 4-3. 포지션 구조
```javascript
{
  market: 'KRW-BTC',
  symbol: 'BTC',
  strategyName: 'RSI반전',
  stages: [
    { price: 50000, qty: 0.1, amount: 5000, time: ... },
    { price: 49000, qty: 0.2, amount: 10000, time: ... }
  ],
  avgPrice: 49333,
  totalQty: 0.3,
  totalInvested: 15000,
  currentStage: 2,
  maxStage: 3  // 잔고 기반 자동 계산
}
```

## 5. 익절/손절

### 5-1. 익절
- **조건**: 현재가 ≥ 평단가 × 1.015 (+1.5%)
- **동작**: 전량 매도
- **수수료**: 왕복 0.08% (FEE_ROUND = 0.0008) 감안하여 실질 수익 = 1.5% - 0.08% ≈ 1.42%

### 5-2. 손절
- **조건**: 최종 물타기 단계 소진 후 평단가 대비 -5%
- **동작**: 전량 매도 (손절)
- 물타기 단계가 남아있으면 손절하지 않고 추가 매수
- **하드 스탑**: 물타기 단계와 무관하게, 포지션 총 미실현 손실이 총자본의 -5% 초과 시 강제 청산

### 5-3. 시간 제한
- **기본 보유**: 60분 + (물타기 단계 × 30분)
  - 1단계: 최대 90분, 2단계: 120분, 3단계: 150분...
- **연장 조건**: 현재 수익 중 (pct > 0) 이면 추가 30분 연장
- 타임아웃 시 현재가로 전량 매도

## 6. 리스크 관리

- **동시 운영 코인**: 최대 2개 (코인별 독립 물타기 스테이지)
- **일일 MDD**: -3% (오늘 실매매 기준, lastLiveTradeTime 기반 리셋)
- **자본 하한**: -10% (초기 자본 대비)
- **일일 최대 매매**: 60회 (각 API 주문 1건 = 1회)
- **코인당 물타기 최대 단계**: 가용잔고 기반 자동 계산
- **포지션 하드 스탑**: 단일 포지션 미실현 손실 > 총자본 5% 시 강제 청산
- **key.json**: autoTrade=true 필수
- **API Key**: 설정 필수
